# forgia-example-todo

> A Grafana TODO app built entirely via [Forgia](https://github.com/Deepzima/forgia) spec-driven development.

## What is this?

This repo demonstrates the full Forgia workflow — from Feature Design to running code, with zero manual coding. Every file was generated by AI agents following structured execution specs (SDDs).

## Stack

- **Backend**: Go + [grafana-app-sdk](https://github.com/grafana/grafana-app-sdk)
- **Frontend**: TypeScript + React (Grafana plugin)
- **Infrastructure**: Kubernetes CRD, Tiltfile, zima-lab deploy
- **Framework**: [Forgia](https://github.com/Deepzima/forgia) v0.6.0

## Workflow

```
/fd-new "TODO Grafana App"
  ↓
/fd-review FD-001 (5 fails → fix → 21/21 pass)
  ↓
/fd-sdd FD-001 (generates 4 SDDs)
  ↓
forgia validate FD-001 (4/4 valid)
  ↓
forgia exec SDD-001 → CRD schema + Go/TS codegen
forgia exec SDD-002 → Backend operator + CRUD handlers
forgia exec SDD-003 → Frontend Grafana plugin
forgia exec SDD-004 → K8s deployment + zima-lab
```

## SDDs

| SDD | Scope | Status | Runner |
|-----|-------|--------|--------|
| SDD-001 | CRD schema, grafana-app-sdk kind (Go + CUE) | Done | claude |
| SDD-002 | Backend operator, CRUD handlers, REST API (Go) | Done | claude |
| SDD-003 | Frontend Grafana plugin (TypeScript + React) | In Progress | claude |
| SDD-004 | K8s manifests, Dockerfile, CI/CD, zima-lab deploy | Planned | — |
