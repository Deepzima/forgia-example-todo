# forgia-example-todo

> A Grafana TODO app built entirely via [Forgia](https://github.com/Deepzima/forgia) spec-driven development.
> **Zero manual coding** — every file generated by AI agents following structured execution specs.

## Demo

![Forgia Demo — FD → SDD generation](docs/forgia-demo.gif)

## What is this?

This repo demonstrates the full Forgia workflow end-to-end: Feature Design → Review → SDD Generation → Validation → Agent Execution. The result is a complete Grafana app with CRD, backend operator, frontend plugin, and K8s deployment — all produced by `forgia exec`.

## Stack

- **Backend**: Go + [grafana-app-sdk](https://github.com/grafana/grafana-app-sdk)
- **Frontend**: TypeScript + React (Grafana plugin)
- **Infrastructure**: Kubernetes CRD, Tiltfile, zima-lab deploy
- **Framework**: [Forgia](https://github.com/Deepzima/forgia) v0.6.0

## Commands Used (full session)

### 1. Setup

```bash
# Install Forgia CLI and Claude Code commands
cd ~/zimafiles/dev/github/zima/forgia
mise run claude:install

# Initialize the project
cd ~/zimafiles/dev/github/zima/forgia-example-todo
forgia init

# Verify
forgia doctor
forgia status
```

### 2. Feature Design

```bash
# Inside Claude Code (claude CLI)
/fd-new "TODO Grafana App — grafana-app-sdk CRUD with zima-lab deploy"

# Review gate — first attempt: 5 checks failed
# (missing solutions, empty interfaces, no Mermaid, generic criteria)
/fd-review FD-001

# Claude fixes all 5 issues, re-runs review
# → 21/21 checks passed → status: approved
```

### 3. SDD Generation

```bash
# Generate 4 execution specs from the approved FD
/fd-sdd FD-001

# Validate all SDDs before execution
forgia validate FD-001
# → Total: 4 | Valid: 4 | Invalid: 0
```

### 4. Agent Execution (FD-001 — manual exec)

```bash
# Execute each SDD sequentially (runner: claude, Max subscription)
forgia exec .forgia/sdd/FD-001/SDD-001-todo-custom-resource.md      # ~7 min
forgia exec .forgia/sdd/FD-001/SDD-002-todo-backend-operator.md     # ~8 min
forgia exec .forgia/sdd/FD-001/SDD-003-todo-frontend-plugin.md      # ~13 min
forgia exec .forgia/sdd/FD-001/SDD-004-deployment-zima-lab.md       # ~5 min
```

### 5. Second Feature (FD-002 — watch mode)

```bash
# Terminal 1: start watcher BEFORE generating SDDs
forgia watch FD-002

# Terminal 2: inside Claude Code
/fd-new "Add priority field to Todo"
/fd-review FD-002       # fix → re-review → approved
/fd-sdd FD-002          # generates 3 SDDs

# Terminal 1 auto-detects and executes:
#   SDD-001 (data model)  → 2m 31s, 14 tests
#   SDD-002 (API layer)   → 3m 55s, 44 tests
#   SDD-003 (UI)          → 4m 51s, 47 tests
```

### 6. Final Status

```bash
forgia status
# 2 FDs, 7 SDDs (all done)
```
